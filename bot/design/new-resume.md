# /new_resume — Design Document

> Статус: ❌ Не реализовано
> Источник: концепция из обсуждения

---

## Назначение

Создание ATS-friendly PDF-резюме на основе LinkedIn-профиля, адаптированного под конкретную вакансию.

---

## Флоу (концепция, 5 шагов)

```
/new_resume
└── [1] Запрос описания вакансии
    └── [2] Анализ вакансии (n8n: извлечение требований)
        └── [3] Matching: сопоставление опыта/навыков пользователя с требованиями
            └── [4] Генерация адаптированного резюме (GPT-4o)
                └── [5] PDF delivery → пользователю в Telegram
```

---

## Ключевые требования

- ATS-friendly формат (без сложных таблиц, шрифт readable, ключевые слова из вакансии)
- Язык: RU / EN (по выбору пользователя или язык вакансии)
- Основа: LinkedIn-профиль пользователя (уже собранный в /test_linkedin)
- Адаптация: переформулировка bullet points опыта под ключевые слова вакансии

---

## Входные данные

- `normalized_profile` из предыдущего аудита (или новый сбор профиля)
- `job_description` — текст вакансии (вставка пользователем)

---

## Выходные данные

- PDF-резюме (через n8n → PDF generation)
- Отправляется пользователю через Telegram file

---

## Связь с монетизацией

- Платная функция (включена в Pro tier или отдельная покупка)
- Стоимость: отдельный upsell после аудита или standalone

---

## Backend задачи (не спроектированы)

- Промпт для анализа вакансии → извлечение: требования, стек, soft skills, уровень
- Промпт для matching: какие части профиля релевантны, что нужно переформулировать
- Промпт для генерации резюме: структура, bullet points, summary
- n8n workflow для PDF generation
- API endpoint: `POST /webhook/linkedin/resume`
