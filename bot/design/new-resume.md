# /new_resume — Design Document

> Статус: ❌ Не реализовано
> Источник: маркетинговый экспорт 2026-02-27
> Цена: 799 Stars (апсейл после Full Report)

---

## Назначение

Создание ATS-оптимизированного PDF-резюме под конкретную вакансию на основе LinkedIn-профиля пользователя.

---

## Флоу бота (5 шагов)

```
/new_resume
└── [1] Запрос описания вакансии
│       "Вставьте текст вакансии (или её ключевые требования)"
│
└── [2] Подтверждение языка
│       "На каком языке сделать резюме? RU / EN"
│
└── [3] Выбор стиля
│       "Compact / Modern / Corporate"
│       (влияет на форматирование PDF, не на контент)
│
└── [4] Обработка (n8n):
│       ├── Анализ вакансии:
│       │   - Выделение hard skills / soft skills
│       │   - Must-have keywords
│       │   - Уровень позиции
│       ├── Matching с профилем:
│       │   - Извлечение релевантного опыта
│       │   - Переформулировка под требования
│       │   - Подбор ключевых слов из вакансии
│       └── Генерация (GPT-4o):
│           - ATS-friendly структура (без таблиц, без графики)
│           - Bullet points опыта переписаны под вакансию
│           - Summary адаптирован под роль
│
└── [5] Доставка в Telegram:
        - PDF файл
        - Текстовая версия (для копипаста)
        - Список добавленных ключевых слов
```

---

## Ключевые требования к выходному резюме

- ATS-friendly: без сложных таблиц, без графики, без колонок
- Шрифт: readable (Arial / Roboto / Helvetica)
- Ключевые слова из вакансии включены органично
- Язык: RU или EN (по выбору)
- Структура: Contact → Summary → Experience → Education → Skills

---

## Входные данные

| Поле | Источник |
|---|---|
| `normalized_profile` | Из предыдущего аудита в `artifacts` PostgreSQL |
| `job_description` | Вставка пользователем в боте |
| `resume_lang` | Выбор в боте (RU/EN) |
| `resume_style` | Выбор в боте (Compact/Modern/Corporate) |

---

## Выходные данные

- PDF-резюме (n8n → PDF generation → Telegram file)
- Текстовый вариант (markdown или plain text)
- Список ключевых слов, добавленных в резюме

---

## Связь с монетизацией

- Апсейл 799 Stars после получения Full Report
- Отдельная покупка: 799 Stars standalone
- После покупки — 1 резюме; повтор = новая покупка

## Backend задачи (не спроектированы)

- Промпт для анализа вакансии (извлечение требований → JSON)
- Промпт для matching (профиль × вакансия → что релевантно)
- Промпт для генерации резюме (ATS-структура + адаптированные bullets)
- n8n workflow: `resume_generation`
- PDF generation: через puppeteer / html-pdf / WeasyPrint в n8n
- API endpoint: `POST /webhook/linkedin/resume`
- Хранение готовых PDF (Telegram file_id → таблица `reports` или отдельная)
